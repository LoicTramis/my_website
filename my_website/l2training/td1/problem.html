2) Problème de conception et idées de résolutions

a) Les interfaces/polymorphisme

Pour ma part, un des problèmes de conception se trouve dans l'implémentation des interfaces. En effet, les interfaces présentent dans le programme ne sont pas utiles (un peu pour la javadoc).

Toutes les classes qui implémentent une interface doivent posséder toutes les méthodes déclarées dans l'interface.

De ce fait, les classes qui ne possèdent pas les méthodes d'une autre classe mais qui implémentent la même interface, doivent posséder les mêmes méthodes.

Une solution possible est d'écrire dans chaque classe, la totalité des méthodes présentes dans les classes qui implémentent la même interface.

Par la suite, créer une interface contenant toutes les méthodes de toutes les classes implémentées.

Enfin, Les méthodes qui sont inutilisables dans les classes, renvoient null ou n'implémente aucun code.

Une telle solution pourrait permettre l'utilisation du polymorphisme, afin de faciliter la programmation et les étapes de test.

Ce problème se pose aussi pour une éventuelle classe abstraite pour la classe « Country » afin d'utiliser un seul type Country dans les ArrayList.

b) ArrayList d'ArryLists

De plus, certains pays n'ont aucun point de référence en commun (par exemple le nom, le code ISO2 et ISO3 sont différents sur tous les fichiers). Par conséquent, l'algorithme de recherche basique ne fonctionne pas et ne trouve aucunes informations.

Une hypothétique solution serait de programmer une méthode de comparaison à partir des 3 ou 4 premières lettres du pays par exemple.

Un dernier problème pour la route.

Le fait que le programme utilise plusieurs objets pour afficher les informations (un objet pour le code ISO3, un objet pour la carte de la région du pays, un objet pour le drapeau), empêche la sauvegarde – et le chargement – sérialisable d'un seul et unique objet.

Une solution à ce dilemme est la création d'une autre classe contenant aussi une ArrayList (ou une HashMap) qui contiendrait les éléments de chaque « sous » ArrayList, afin d'obtenir un objet avec toutes les informations relatives aux pays (i.e. le nom, le nom de la région, la population, la carte de la région, le drapeau...).
